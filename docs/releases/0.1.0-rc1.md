# Fork release candidate 0.1.0-rc1

Status: **release candidate for local validation; not published**

This candidate packages the first cohesive release of Maylton's Linux-focused
fork of `turing-smart-screen-python`.

## Highlights

- GTK4/Libadwaita configuration center and media tools.
- Native Rev. C video overlay and storage management.
- Safe process-wide display ownership and shutdown.
- GIF/video preparation with advanced framing and preview.
- Reusable, orientation-aware display profiles.
- Safe automatic display detection with conservative configuration updates.
- Repeatable installer, update flow, and release verification.

## Supported environments

The fork targets Linux desktop environments and Raspberry Pi systems capable
of running the inherited Python application and its dependencies. Python 3.9
or newer is required; local release validation includes Python 3.14.

Windows and macOS remain part of the inherited upstream codebase, but the
fork-specific GTK, desktop integration, packaging, and release workflow are
Linux-focused and are not claimed as validated on those platforms.

## Hardware validation

The following native-media workflow has been physically validated:

- Turing Smart Screen Rev. C;
- 2.1-inch, 480 × 480;
- ROM 88 baseline;
- H.264 in MP4 with `yuv420p`;
- upload, listing, playback, stop, size inspection, and deletion.

The inherited system-monitor functionality supports additional revisions and
sizes, but that does not imply validation of fork-specific native video or
storage commands.

## Experimental and preview-only functionality

The media editor can prepare and preview output for additional square,
portrait, landscape, and ultrawide profiles. Native upload remains disabled
for these profiles until the matching hardware and firmware are validated.

Automatic detection is conservative. Shared or ambiguous identifiers require
manual confirmation, and Rev. C physical size is preserved from the active
theme when it cannot be determined reliably.

## Known limitations

- Native media commands are validated only on Rev. C 2.1-inch ROM 88.
- Output-size estimation is advisory and varies with source complexity.
- FFmpeg and ffprobe are required for media preparation.
- GTK4/Libadwaita availability depends on the Linux distribution.
- Some devices expose more than one serial endpoint.
- Automatic detection cannot safely identify every physical size or clone.
- No signed packages, release tag, or GitHub Release are produced by this
  candidate.

## Installation

```bash
git clone https://github.com/maylton/turing-smart-screen-video-overlay.git
cd turing-smart-screen-video-overlay
./install.sh
```

For desktop autostart:

```bash
./install.sh --autostart
```

## Release verification

From a development checkout with its virtual environment installed:

```bash
scripts/verify-release-readiness.sh
```

The verifier checks metadata, documentation, shell and Python syntax, unit
tests, whitespace, and optional real-FFmpeg integration tests.
